The most efficient way to receive support is to open an issue_ on GitHub or post in the `official support thread`_ on the XDA forum.

Pre-support checklist
=====================

To save time, please complete these steps before reaching out:

1. **Read the docs:** Review the `installation instructions <./INSTRUCTIONS.rst>`_ thoroughly.
2. **Search issues:** Check `existing issues <issue_>`_ to see if your problem has already been addressed.
3. **Browse XDA:** Search the `official support thread`_ for known workarounds or similar cases.

Reporting a bug
===============

When opening a new issue, please provide as much of the following information as possible:

* **Issue description:**
   - A clear summary of the problem.
   - Specific steps to reproduce the error.

* **System details:**
   - Installer version (e.g., ``1.3.1-beta``).
   - Device brand and model.
   - Android version.
   - Name and version of the ROM.
   - Installation method (manual flash via recovery, ADB sideload or ``zip-install.sh``).
   - Recovery version (if flashing via recovery).

* **Diagnostic data:**
   - **Debug log:** Attach the log file (named ``debug-a5k.log``) which is usually located on the microSD card or provide the on-screen output.

*The more detail you provide, the faster we can investigate and resolve your issue.*
